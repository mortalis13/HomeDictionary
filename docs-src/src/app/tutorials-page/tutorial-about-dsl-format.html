About the DSL Dictionary Format

The source text of a DSL dictionary is a plain text file saved with the .dsl extension.

A dictionary structure includes the header and dictionary entries.

The header contains the dictionary name, its translation direction (source - target languages) and other information.

The entries are pairs of headwords and their translations composed using a special markup language.


An example structure:


  #NAME "Some Dictionary"
  #INDEX_LANGUAGE "English"
  #CONTENTS_LANGUAGE "German"

  aggregate
    [m1]1. (sich) anhäufen[/m]
    [m1]vereinigen ([b]to[/b] mit)[/m]
    [m1]sich belaufen auf ([p]Akk[/p])[/m]
    [m1]2. (an)gehäuft[/m]
    [m1]gesamt[/m]
    [m1]3. Anhäufung [p]f[/p][/m]
    [m1]Gesamtmenge [p]f[/p], Summe [p]f[/p][/m]
    [m1]Aggregat [p]n[/p][/m]
  therapy
    [m1]Therapie [p]f[/p][/m]

The tags are used inside the entry body to apply different styles to different parts of the text (indent paragraphs depending on their level, make text bold-italic, create links to other articles, change text color).


The following DSL tags are supported:

[b]bold
[i]italics
[u]underline
[c]color
[mN]paragraph of level N
[ex]example
[com]comment
[s]multimedia (picture, sound)
[url] web link
[p]label
[ref]link to another card
[sub]subscript
[sup]superscript
[*]zone for the full translation mode (in Lingvo app)

All tags are closed using the [/TAG] format, [m] paragraph tags are closed with [/m].


DSL files are used directly by some applications (like GoldenDict) or need to be compiled into a compressed LSD format to be used in the ABBYY Lingvo application.


A more extended example of a DSL file: github.com/Tvangeste/SampleDSL/blob/master/sample.dsl.

More information about the DSL format and its compilation: lingvo.helpmax.net/en/troubleshooting/dsl-compiler.
